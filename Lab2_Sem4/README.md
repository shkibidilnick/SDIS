**Тема:** разработка оконного приложения с графическим интерфейсом на Python

## 1. Цель работы

Целью лабораторной работы является изучение принципов построения графического пользовательского интерфейса и разработка программной системы с GUI на языке Python в соответствии с заданием.

В рамках работы необходимо было разработать приложение, которое:
- построено по шаблону **Model-View-Controller**;
- имеет **главное окно** и **дочерние диалоги**;
- позволяет **добавлять**, **искать**, **удалять**, **отображать**, **сохранять** и **загружать** записи;
- хранит записи в **базе данных**;
- поддерживает экспорт и импорт данных в **XML**;
- показывает записи как в виде **таблицы**, так и в виде **дерева**;
- поддерживает **постраничный вывод**;
- покрыта **unit-тестами**.

---

## 2. Постановка задачи

Согласно варианту 8, каждая запись содержит следующие поля:
- имя питомца;
- дата рождения;
- дата последнего приёма;
- ФИО ветеринара;
- диагноз.

Необходимо реализовать поиск и удаление записей по следующим условиям:
1. по имени питомца и дате рождения;
2. по дате последнего приёма и ФИО ветеринара;
3. по фразе из диагноза.

Также приложение должно обеспечивать:
- ввод записей через отдельное диалоговое окно;
- вывод всех текущих записей в главном окне;
- вывод результатов поиска в отдельном диалоге поиска;
- удаление записей через отдельный диалог удаления;
- уведомление пользователя о результате удаления;
- сохранение и загрузку записей из XML-файла;
- хранение данных в базе SQLite;
- альтернативное отображение данных в виде дерева;
- постраничный вывод записей.

## 4. Архитектура приложения

Приложение реализовано по шаблону **MVC (Model-View-Controller)** с пассивной Model.

### 4.1. Model

Слой Model включает:
- доменные сущности;
- критерии поиска и удаления;
- валидаторы;
- пагинацию;
- работу с репозиторием;
- XML-импорт и XML-экспорт.

Model не зависит от графического интерфейса и содержит логику предметной области.

### 4.2. View

Слой View включает:
- главное окно программы;
- диалог добавления записи;
- диалог поиска;
- диалог удаления;
- таблицу записей;
- дерево записей;
- виджет пагинации;
- сообщения пользователю.

View отвечает только за отображение данных и ввод информации пользователем.

### 4.3. Controller

Слой Controller связывает интерфейс и бизнес-логику:
- обрабатывает действия пользователя;
- вызывает методы сервисов;
- обновляет представление после операций добавления, поиска, удаления, импорта и экспорта.

## 6. Описание основных модулей

### 6.1. Доменные сущности

В системе используются основные сущности:
- `PersonName` — ФИО ветеринара;
- `PetRecord` — запись о питомце и последнем приёме.

Для корректности ввода реализованы проверки:
- имя питомца не должно быть пустым;
- диагноз не должен быть пустым;
- дата рождения не может быть позже текущей даты;
- дата последнего приёма не может быть раньше даты рождения;
- ФИО ветеринара должно быть задано корректно.

### 6.2. Репозиторий

Для хранения данных используется репозиторий `SQLitePetRecordRepository`, который обеспечивает:
- создание таблицы в базе данных;
- добавление записей;
- получение всех записей;
- удаление записей по условиям варианта;
- поиск по условиям варианта;
- полную замену набора записей при импорте из XML.

### 6.3. Сервисы

В проекте реализованы сервисы:
- `PetRecordService` — добавление записей и получение страниц данных;
- `SearchService` — поиск записей;
- `DeleteService` — удаление записей;
- `ImportExportService` — импорт и экспорт XML.

### 6.4. XML-модуль

Для обмена данными реализованы:
- `PetRecordDomWriter` — запись XML через DOM;
- `PetRecordSaxReader` — чтение XML через SAX.

Это соответствует требованиям задания.

## 7. Функциональные возможности системы

Разработанное приложение поддерживает следующие функции:

### 7.1. Добавление записи
Пользователь открывает диалог добавления, вводит:
- имя питомца;
- дату рождения;
- дату последнего приёма;
- ФИО ветеринара;
- диагноз.

После подтверждения запись проходит валидацию, сохраняется в базе данных и отображается в главном окне.

### 7.2. Отображение записей
Главное окно показывает записи:
- в виде таблицы;
- в виде дерева, где каждая запись раскрывается по отдельным полям.

### 7.3. Поиск записей
В отдельном диалоге поиска пользователь может искать записи:
- по имени питомца и дате рождения;
- по дате последнего приёма и ФИО ветеринара;
- по фразе из диагноза.

Результаты поиска выводятся в этом же диалоговом окне.

### 7.4. Удаление записей
В отдельном диалоге удаления пользователь задаёт условия удаления:
- по имени питомца и дате рождения;
- по дате последнего приёма и ФИО ветеринара;
- по фразе из диагноза.

После удаления система сообщает пользователю, сколько записей удалено, либо сообщает, что записи не были найдены.

### 7.5. Работа с XML
Реализованы:
- экспорт текущих записей в XML-файл;
- импорт записей из XML-файла.

### 7.6. Пагинация
Для удобства просмотра записей реализован постраничный вывод с возможностью:
- перехода на первую страницу;
- перехода на предыдущую страницу;
- перехода на следующую страницу;
- перехода на последнюю страницу;
- изменения количества записей на странице.
