# Lab2_Sem4

Лабораторная работа №2 по дисциплине «Проектирование программного обеспечения интеллектуальных систем».

Проект реализует оконное приложение на **PySide6** для учёта приёмов питомцев в ветеринарной клинике.

## Что реализовано

- MVC с пассивной Model
- главное окно, меню, toolbar, строка состояния
- дочерние диалоги: добавление, поиск, удаление
- отображение записей в таблице
- альтернативное представление записей в виде дерева
- пагинация с переходом на первую/предыдущую/следующую/последнюю страницу
- хранение в SQLite
- экспорт в XML через DOM
- импорт из XML через SAX
- unit-тесты для доменной логики, репозитория, XML и сервисов
- демонстрационные XML-файлы по 50+ записей

## 1. Что сделать в PyCharm с самого начала

### Связать проект с вашим GitHub-репозиторием

У вас репозиторий: `https://github.com/shkibidilnick/SDIS`

1. Откройте **PyCharm**.
2. Выберите **Get from VCS** или **File -> New Project from Version Control**.
3. Вставьте URL репозитория:
   `https://github.com/shkibidilnick/SDIS`
4. Клонируйте его на компьютер.
5. Внутри репозитория создайте папку проекта **Lab2_Sem4**.
6. Откройте именно папку `SDIS` как корневую папку проекта в PyCharm.
7. Убедитесь, что структура выглядит так:

```text
SDIS/
└── Lab2_Sem4/
```

### Инициализация виртуального окружения

В терминале PyCharm, находясь в `SDIS/Lab2_Sem4`, выполните:

```bash
python -m venv .venv
```

## 4. Архитектура проекта

```text
app/
  controllers/       # контроллеры
  domain/            # сущности и критерии
  repositories/      # SQLite-репозиторий
  services/          # бизнес-сервисы
  views/             # GUI
  xmlio/             # DOM writer + SAX reader
  main.py            # точка входа

tests/               # unit-тесты
sample_data/         # демонстрационные XML-файлы
```

## 5. Сценарий демонстрации

1. Запустить приложение.
2. Добавить несколько записей через диалог.
3. Выполнить поиск по имени питомца и дате рождения.
4. Выполнить поиск по дате последнего приёма и ФИО ветеринара.
5. Выполнить поиск по фразе из диагноза.
6. Удалить записи по одной из групп условий.
7. Показать таблицу и дерево.
8. Экспортировать данные в XML.
9. Очистить/изменить БД и затем загрузить XML обратно.
10. Показать постраничный вывод.

## 6. Замечания

- Даты хранятся как `date`, а не как строки.
- XML-запись выполняется через DOM.
- XML-чтение выполняется через SAX.
- Основная часть тестов сосредоточена на слоях domain/services/repositories/xmlio, чтобы добиться высокого покрытия без хрупких GUI-тестов.
