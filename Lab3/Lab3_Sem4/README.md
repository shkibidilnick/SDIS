## Описание игры и правила

Игрок управляет водопроводчиком Марио, который должен пройти уровень слева направо до флага в правом конце карты. По дороге встречаются:

- **враги** трёх типов:
  - **Гумба** (бурая) — ходит туда-сюда по платформам, разворачивается у стен и обрывов;
  - **Купа** (зелёная) — умная, не падает в пропасть;
  - **Пиранья** (зелёное растение с красной головой) — выезжает из труб циклически: прячется → вылезает → видна → прячется обратно;
- **монеты** (символ `C` на карте) — дают очки, каждые 50 монет дают **+1 жизнь**;
- **блоки `?`** — содержат случайный бонус;
- **усилители (powerup)** трёх типов:
  - **Гриб** (красный с белыми пятнами) — превращает Марио в большого. Большой Марио держит один удар, ломает кирпичи прыжком снизу. Длительность **20 секунд**, после чего возвращается в маленького с временной неуязвимостью;
  - **Звезда** (золотая, прыгает) — даёт неуязвимость на **10 секунд**;
  - **Огненный цветок** (оранжевый) — даёт ускорение на **12 секунд**.

**Цель:** дойти до флага, набрав максимум очков.

**Жизни:** в начале даётся 3 жизни. Жизнь теряется при касании врага в маленькой форме, при падении в пропасть или при истечении времени. Когда жизней не остаётся — Game Over.

**Ограничение по времени:** 300 секунд. За оставшееся время по окончании уровня даются бонусные очки.

### Взаимодействие с врагами

| Состояние Марио | Гумба | Купа | Пиранья |
|---|---|---|---|
| Маленький, прыжок сверху | Гумба гибнет | Купа гибнет | (нельзя топтать) |
| Маленький, контакт сбоку | Смерть | Смерть | Смерть |
| Большой, контакт сбоку | Гумба гибнет, Марио остаётся big | Марио → small | Марио → small |
| Со звездой (маленький размер) | Марио не гибнет при контакте сбоку | Марио не гибнет при контакте сбоку | Марио не гибнет при контакте сбоку |
| Со звездой (Большой размер) | Гумба гибнет от касания | Купа гибнет от касания | Пиранья гибнет от касания |

### Очки

| Действие | Очки |
|---|---|
| Подобрать монету | 50 |
| Победить Гумбу (прыжком сверху) | 100 |
| Победить Купу | 200 |
| Победить Пиранью (звездой) | 200 |
| Подобрать гриб | 200 |
| Подобрать огненный цветок | 300 |
| Подобрать звезду | 500 |
| Завершить уровень | 1000 |
| Бонус за оставшееся время | 10 / сек |
| Бонус за каждую собранную монету | 5 / шт |

Если в конце игры набранный счёт **попадает в топ-5**, появляется диалог ввода имени, и результат сохраняется в `config/highscores.json`.

### Распределение бонусов из `?`-блоков

Содержимое блока выбирается случайно при ударе:
- **Гриб** — 40%;
- **Огненный цветок** — 35%;
- **Звезда** — 25%.

## Управление

| Клавиша | Действие |
|---|---|
| `←` / `A` | Идти влево |
| `→` / `D` | Идти вправо |
| `Space` / `↑` / `W` | Прыжок |
| `Shift` (зажат) | Бег (повышенная скорость) |
| `Enter` | Подтвердить выбор в меню |
| `Esc` | Пауза / выход в меню |

В меню навигация стрелками `↑`/`↓`, выбор — `Enter`.

### UX-улучшения прыжка

Реализованы два «прощающих» механизма для отзывчивого управления:

- **Jump buffer (6 кадров):** если игрок нажал прыжок чуть-чуть до приземления, прыжок всё равно сработает при касании земли.
- **Coyote time (6 кадров):** если игрок только что слетел с платформы, прыжок ещё разрешён в течение 6 кадров.

Это компенсирует задержки восприятия и аппаратные ограничения клавиатуры.

## Архитектура проекта

Проект построен на трёх слоях.

### 1. Конфигурация (`config/`)

JSON-файлы, описывающие все игровые сущности декларативно. Изменение игрового баланса не требует правки кода.

- `settings.json` — экран, физика, очки, аудио, тайминги;
- `level_map.json` — карта уровня (массив строк-символов 250×19);
- `enemies.json` — типы врагов и точки спавна;
- `powerups.json` — типы бонусов и их параметры;
- `highscores.json` — таблица рекордов (топ-5).

### 2. Утилиты (`src/utils/`) — слой инфраструктуры

| Модуль | Ответственность |
|---|---|
| `config_loader.py` | Загрузка/сохранение JSON, кеширование |
| `geometry.py` | Класс `Rect` для AABB-коллизий (без зависимости от pygame) |
| `highscore_manager.py` | Чтение/запись рекордов, проверка попадания в топ-5 |
| `audio_manager.py` | Программный синтез WAV (синус/квадрат/пила), управление миксером |
| `renderer.py` | Программное рисование спрайтов и интерфейса |

### 3. Сущности и сцены (`src/entities/`, `src/scenes/`) — игровая логикаPhysicsEntity (база: гравитация + AABB-коллизии)
```text
├── Player        (small/big, big_timer, бонусы, jump buffer, coyote time)
├── Enemy         (поведения: walk, walk_smart, piranha)
├── PowerUp       (анимация эмерджа, движение, разворот от стен)
├── Particle      (частицы для анимаций)
└── Level         (карта тайлов, поиск твёрдых/интерактивных)
```

Сцены реализуют шаблонный метод (базовый класс `Scene` с `handle_events`/`update`/`draw`):MenuScene → GameScene → NameEntryScene → 
```text
HighScoresScene → MenuScene
↓
HelpScene
```

Каждая сцена устанавливает `next_scene` и `done = True`, а `App` (`src/app.py`) переключает их.

### Поток событий
```text
pygame.event.get()  →  scene.handle_events(events)  →  scene.update(dt)  →  scene.draw(surface)
```

### Диаграмма зависимостей модулей
```text
main.py
├─ src.app.App

├─ src.entities.* (Player/Enemy/PowerUp/Particle/Level)

└─ src.utils.geometry
```

## Конфигурация

Все игровые настройки можно менять без правки кода в файлах `config/*.json`.

**Изменить количество стартовых жизней:** в `settings.json`:
```json
"player": { "start_lives": 5 }
```

**Изменить силу прыжка:** в `settings.json`:
```json
"physics": { "jump_strength": -15 }
```

**Добавить врага:** в `enemies.json`, секция `spawns`:
```json
{ "type": "goomba", "x": 50, "y": 14 }
```

**Добавить пиранью в трубу:** координата = верх трубы:
```json
{ "type": "piranha", "x": 14, "y": 12 }
```

**Изменить длительность бонуса** (значения в кадрах при 60 FPS):
```json
{ "type": "star", "duration": 600 }
```

## Принципы разработки

- **SRP:** каждый класс выполняет одну задачу.
- **OCP / Strategy через конфиг:** новый враг или бонус добавляется правкой JSON.
- **DRY:** общая физика вынесена в `PhysicsEntity`.
- **KISS:** спрайты и звуки генерируются программно.
- **YAGNI:** реализовано только то, что нужно по ТЗ.
- **Декаплинг от pygame:** `geometry.Rect` и физика тестируются без графики.
