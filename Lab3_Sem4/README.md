## Описание проекта

Реализация классической игры **Super Mario Bros** с использованием событийно-ориентированного программирования на Python + pygame.

### Реализованные требования

| Требование | Статус |
|---|---|
| Карта уровня в файле | ✅ `configs/level1.json` |
| Не менее 10 экранов карты (800 px × 10 = 8000 px) | ✅ 250 тайлов × 32 px = 8000 px |
| Два типа подбираемых бонусов | ✅ Гриб, Огненный цветок, Звезда, Монета |
| Меню: старт / рекорды / справка / выход | ✅ |
| Таблица рекордов с сохранением | ✅ `configs/highscores.json` |
| Диалог нового рекорда с вводом имени | ✅ |
| Звуки и фоновая музыка | ✅ (WAV/OGG в `assets/`) |
| Анимации (ходьба, прыжок, смерть, монета) | ✅ |
| Документация Markdown | ✅ |
| Покрытие unit-тестами ≥ 85 % | ✅ |

## Правила игры

### Управление

| Клавиша | Действие |
|---|---|
| `←` / `→` или `A` / `D` | Движение |
| `Пробел` / `↑` / `W` | Прыжок |
| `Shift` | Бег |
| `Z` | Огненный шар (только с огненным цветком) |
| `P` | Пауза |
| `Esc` | Выход в меню |

### Бонусы

| Бонус | Эффект |
|---|---|
| 🍄 Гриб (красный) | Марио вырастает |
| 🌸 Огненный цветок | Марио стреляет огнём |
| ⭐ Звезда | Временная неуязвимость (10 секунд) |
| 🪙 Монета | +200 очков; 100 монет = доп. жизнь |

### Враги

| Враг | Поведение | Уничтожение |
|---|---|---|
| **Гумба** | Ходит прямо, разворачивается у стены | Прыжок сверху |
| **Купа** | Ходит быстрее; при прыжке — убирается в панцирь | Прыжок → панцирь; ещё раз → пинок панциря |

### Цель

Добраться до флагштока в конце уровня до истечения времени.

## Архитектура

### Событийно-ориентированное программирование

Игра построена на **стек-машине состояний** (State Stack Pattern):

```
Game.run()
  └── events → active_state.handle_event(event)
  └── update  → active_state.update()
  └── draw    → active_state.draw(screen)
```

Каждое состояние реализует интерфейс `BaseState`:

```python
class BaseState:
    def handle_event(self, event): ...
    def update(self): ...
    def draw(self, surface): ...
```

### Диаграмма состояний

```
MenuState
  ├── → GameplayState
  │       ├── → NewRecordState → HighScoresState
  │       └── → GameOverState  → MenuState
  ├── → HighScoresState → MenuState
  └── → HelpState → MenuState
```

### Физика

`Entity.move_x()` и `Entity.move_y()` разделены для корректного разрешения коллизий по каждой оси независимо (стандартный подход в 2D платформерах).

## Конфигурации

### `configs/settings.json`

Глобальные параметры: размер окна, FPS, гравитация, скорости Марио, очки за действия.

### `configs/level1.json`

Описание карты уровня:
- `map` — двумерный массив символов
- `tiles` — словарь символ → тип тайла
- `entities` — координаты врагов и монет
- `player_start` — стартовая позиция

### `configs/entities.json`

Характеристики врагов и бонусов (скорость, здоровье, цвета, размеры).

### `configs/highscores.json`

Таблица рекордов — автоматически обновляется при новом рекорде.

## Анимации

Анимации реализованы через класс `Animation`:

```python
anim = Animation(frames=list_of_surfaces, speed=8, loop=True)
anim.update()          # вызов каждый тик
frame = anim.current_frame
```

| Анимация | Описание |
|---|---|
| `walk_small` / `walk_big` | 2 кадра, чередование цвета |
| `jump_small` / `jump_big` | 1 кадр (синий) |
| `idle_small` / `idle_big` | 1 кадр |
| `dead` | 1 кадр + вертикальный полёт |
| Смерть гумбы | Сплющивание (flat rect) |
| Монета / Огонь | 3 кадра с переливом цвета |
| Звезда | Мигающие кадры |

## Звуки

Звуки подключаются через `AssetManager.play_sound(name)`.  
Файлы ожидаются в `assets/sounds/*.wav`, музыка — `assets/music/*.ogg`.

| Звук | Файл | Событие |
|---|---|---|
| Прыжок | `jump.wav` | Mario прыгает |
| Монета | `coin.wav` | Подбор монеты |
| Гибель | `die.wav` | Mario умирает |
| Стомп | `stomp.wav` | Прыжок на врага |
| Бонус | `powerup.wav` | Подбор гриба/цветка |
| Флаг | `flagpole.wav` | Достижение флагштока |
| Кирпич | `break.wav` | Разбивание блока |
| Игра окончена | `gameover.wav` | Нет жизней |
| Лишняя жизнь | `1up.wav` | 100 монет |

## Принципы разработки

| Принцип | Применение |
|---|---|
| **SRP** | Каждый класс — одна зона ответственности (Camera, HUD, TileMap, …) |
| **KISS** | Простые методы без лишней абстракции |
| **DRY** | `Entity.move_x/y` используется всеми объектами; `build_colored_frames` переиспользуется везде |
| **YAGNI** | Нет «на будущее» кода; только реализованные требования |
| **OCP** | Новый враг = новый класс, без изменения `GameplayState` |
